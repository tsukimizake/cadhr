# Code Style
- Before implement something yourself, try to use utility packages I want to keep the code readable. Especially, don't forget to use bevy-async-ecs. Never make a queue yourself.
- If you got lost or think there is any misunderstanding on me, feel free to ask me.
- 過去仕様や撤回した実装、修正したバグについて言及するコメント (例: 「以前は X だったが Y に変えた」「旧 X 版では Z が必要だった」「unify を試みていたが撤回した」など) は絶対に書かないこと。コメントは「コードを読んでも分からない現在の仕様・意図・制約」だけを書くべきで、過去の経緯は git log に任せること。
